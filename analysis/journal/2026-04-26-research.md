## Key Discovery: SwStr% Is Already Available In Our Pipeline

`mlb_data_pipeline.py` (lines 900-906) already pulls Statcast pitch-level data and computes Whiff% (swinging strikes / swings). But SwStr% is swinging strikes / total pitches - different denominator. The fix:

```python
# Current (Whiff%):
whiff_pct = swinging_strikes / swings * 100

# Needed (SwStr%):
swstr_pct = swinging_strikes / total_pitches * 100

# Also add CSW% (Called Strike + Whiff) - best single K predictor:
csw_pct = (called_strikes + swinging_strikes) / total_pitches * 100
```

CSW% (Called Strike + Whiff) is even better than SwStr% for K prediction. We have all the data already.

## Conway-Maxwell-Poisson

Paper: arXiv 2409.17129. Key advantage over NegBin: handles UNDER-dispersion (low-scoring pitching duels) as well as over-dispersion. NegBin only does overdispersion.

For betting: better probability estimates on unders in pitching duels, which is exactly where our game model struggles.

Implementation: no scipy support, needs custom PMF. ~20 lines of code. Slow but fine for ~15 possible scores per game.
